# 🤖 aicoder: AI 네이티브 멀티 에이전트 개발 프레임워크

> [한국어] | [简体中文](README.md) | [繁體中文](README_TW.md) | [English](README_EN.md) | [日本語](README_JP.md)

> **제작자 Sting의 승인을 받은 공식 릴리스**

`aicoder`는 **Claude Code**, **Cursor**, **Windsurf**와 같은 현대적인 AI 코딩 도구를 위해 설계된 협업 개발 스킬셋(Skills)입니다. 전문 소프트웨어 팀의 분업을 시뮬레이션함으로써 복잡한 개발 프로세스를 예측 가능하고 높은 품질의 '게이트 방식' 반복(Iteration)으로 전환합니다. 이를 통해 개발자(특히 초보자)가 처음부터 엔지니어링 표준을 갖춘 프로젝트를 구축할 수 있도록 돕습니다.

---

## 🌟 핵심 컨셉: 멀티 에이전트 협업

`aicoder`는 단순히 코드를 생성하는 기능 그 이상입니다. 완전한 **디지털 팀 아키텍처**를 구축합니다. 명확한 폴더 격리와 역할 정의를 통해 AI는 현재 컨텍스트에 따라 다음 역할 사이를 원활하게 전환할 수 있습니다.

- **🛡️ 워크플로우 수호자 (CLAUDE.md)**: 글로벌 라우팅, 게이트 체크(Gate 1-4) 및 반복 마무리를 담당합니다. 프로젝트가 단계를 건너뛰지 않도록 관리하고 지식 축적이 체계적으로 이루어지도록 보장합니다.
- **🎨 제품 전문가 (product.md)**: IDEO 디자인 씽킹을 기반으로 합니다. 시나리오 재구성, 요구사항 발산 및 수렴, 엄격한 PRD 출력 및 비주얼 사양 수립을 담당합니다.
- **📐 테크 아키텍트 (tech.md)**: 안정성 우선의 의사결정자입니다. 기술 스택 선정, 데이터베이스 모델링, API 설계 및 TRD 작성을 담당합니다.
- **👨‍💻 풀스택 개발자 (developer.md)**: 독립적인 판단력을 가진 장인입니다. 코드 구현, 유닛 체크, 통합 테스트 및 사양 불일치 검토를 담당합니다.

## 🚀 주요 특징

### 1. 게이트 시스템 (Gate-Based Iterations)
프로젝트는 각 단계가 문서화되고 검증되도록 4개의 명확한 게이트로 나뉩니다.
- **Gate 1**: PRD 확정 ('무엇을 만들 것인가' 정의).
- **Gate 2**: TRD 확정 ('어떻게 만들 것인가' 정의).
- **Gate 3**: 개발, 통합 및 인수 완료 ('올바르게 만들어졌는가' 확인).
- **Gate 4**: 반복 마무리 및 지식 축적 ('무엇이 남았는가' 확인).

### 2. 표준화된 문서 체계
모든 반복 작업은 `sprint.md`(작업 목록), `decisions.md`(기술 결정 기록), `backlog.md`(미결 문제)와 같은 표준화된 산출물을 생성합니다. 이를 통해 AI의 긴 세션에서 발생하기 쉬운 '환각(Hallucination)'이나 '망각' 문제를 완벽하게 해결합니다.

### 3. 글로벌 표준 통합
프론트엔드(Vue 3/TS/Vite) 및 백엔드(NestJS/MySQL)를 위한 **코드 품질 레드라인**이 내장되어 있습니다. 반응형 체크리스트와 보안 리뷰를 자동으로 적용하여 프로덕션 레벨의 결과물을 보장합니다.

## 🛠️ 시작하기

1. **워크스페이스 초기화**:
   - 이 프로젝트를 AI 코딩 도구에 로드합니다.
   - "새 프로젝트 [프로젝트명] 초기화해줘"라고 요청합니다.
   - 워크플로우 수호자가 표준 `product/`, `tech/`, `developer/` 디렉토리 구조를 자동으로 설정합니다.

2. **역할 주입**:
   - 각 워크스페이스의 `CLAUDE.md`에는 해당 역할에 대한 지침이 포함되어 있습니다.
   - 특정 폴더에 진입하면 AI는 자동으로 해당 사고 모델을 로드합니다.

## 📂 프로젝트 구조 샘플

```text
aicoder/
├── roles/               # 핵심 역할 지침 (product, tech, developer, coordinator)
├── templates/           # 문서 템플릿 및 코딩 표준 샘플
├── projects/            # 세션 간 공유 지식 베이스 (project.md, decisions, etc.)
├── product/             # 제품 디자인 워크스페이스
├── tech/                # 기술 디자인 워크스페이스
└── developer/           # 코드 구현 및 테스트 워크스페이스
```
