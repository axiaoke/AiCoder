# 🤖 aicoder: AIネイティブ・マルチエージェント開発フレームワーク

> [日本語] | [简体中文](README.md) | [繁體中文](README_TW.md) | [English](README_EN.md) | [한국어](README_KR.md)

> **作成者 Sting による公認リリース**

`aicoder` は、**Claude Code**、**Cursor**、**Windsurf** などの最新の AI コーディングツール向けに設計された、協調開発スキルセット（Skills）です。プロフェッショナルなソフトウェアチームの分業をシミュレートすることで、複雑な開発プロセスを予測可能で高品質な「ゲート式」イテレーションに変換し、開発者（特に初心者）がゼロからエンジニアリング基準を満たすプロジェクトを構築するのを支援します。

---

## 🌟 コアコンセプト：マルチエージェント・コラボレーション

`aicoder` は単なるコード生成ではありません。完全な**デジタルチームアーキテクチャ**を構築します。フォルダ分離と役割定義を明確にすることで、AI は現在のコンテキストに基づいて、以下の役割をシームレスに切り替えることができます。

- **🛡️ ワークフロー守護者 (CLAUDE.md)**: グローバルルーティング、ゲートチェック（Gate 1-4）、およびイテレーションの締め括りを担当します。プロジェクトが段階をスキップしないこと、およびナレッジの蓄積を確実にします。
- **🎨 プロダクトスペシャリスト (product.md)**: IDEO のデザイン思考に基づいています。シナリオの再構成、要件の発散と収束、厳密な PRD の出力、およびビジュアル仕様の確立を担当します。
- **📐 アーキテクト (tech.md)**: 安定性優先の意思決定者です。技術選定、データベースモデリング、API 設計、および TRD の作成を担当します。
- **👨‍💻 フルスタックエンジニア (developer.md)**: 独自の判断力を持つ職人です。コードの実装、ユニットチェック、統合テスト、および仕様の不一致レビューを担当します。

## 🚀 主な特徴

### 1. ゲートシステム (Gate-Based Iterations)
プロジェクトは、各ステップが文書化および検証されることを確実にするために、4つの明確なゲートに分解されます。
- **Gate 1**: PRD 確定（「何を作るか」を定義）。
- **Gate 2**: TRD 確定（「どう作るか」を定義）。
- **Gate 3**: 開発・統合・受け入れ完了（「正しく作られたか」を確認）。
- **Gate 4**: イテレーション完了とナレッジ蓄積（「何が残ったか」を確認）。

### 2. 標準化されたドキュメント体系
各イテレーションでは、`sprint.md`（タスクリスト）、`decisions.md`（技術決定記録）、`backlog.md`（未解決事項）などの標準化された成果物が作成されます。これにより、AI の長いセッションで発生しがちな「ハルシネーション（幻覚）」や「忘却」の問題を解決します。

### 3. グローバル基準の統合
フロントエンド（Vue 3/TS/Vite）およびバックエンド（NestJS/MySQL）向けの**コード品質レッドライン**を内蔵。レスポンシブ・チェックリストやセキュリティレビューを自動的に適用し、プロダクションレベルの出力を保証します。

## 🛠️ はじめかた

1. **ワークスペースの初期化**:
   - このプロジェクトを AI コーディングツールに読み込みます。
   - 「新しいプロジェクト [プロジェクト名] を初期化して」と伝えます。
   - ワークフロー守護者が、標準的な `product/`, `tech/`, `developer/` ディレクトリ構造を自動的にセットアップします。

2. **役割の注入**:
   - 各ワークスペースの `CLAUDE.md` には、対応する役割の指示が含まれています。
   - フォルダに移動すると、AI は自動的に対応する思考モデルをロードします。

## 📂 プロジェクト構造のサンプル

```text
aicoder/
├── roles/               # コアな役割の指示 (product, tech, developer, coordinator)
├── templates/           # ドキュメントテンプレートとコーディング規約のサンプル
├── projects/            # セッションを跨いだ共有ナレッジベース (project.md, decisions, etc.)
├── product/             # プロダクトデザイン・ワークスペース
├── tech/                # テクニカルデザイン・ワークスペース
└── developer/           # コード実装・テスト用ワークスペース
```
